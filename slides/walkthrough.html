and: Conjunción

La operación lógica de conjunción se expresa en Python usando la palabra and. Si se tiene la expresión p and q se sabe entonces que tendrá un valor verdadero sólo si la expresión p y la expresión q tienen simultáneamente un valor verdadero. Además, el operador and es asociativo, así que se pueden escribir expresiones como w and x and y and z sin tener que usar paréntesis y sin riesgos de que cambie el resultado.

Pequeña diferencia

La diferencia entre el operador and de Python y la operación de conjunción es que Python se aprovecha de las propiedades de identidad y dominancia de la conjunción ( p∧Verdadero=p y *p∨Falso=Falso) para evitar calcular más términos de los necesarios.

Esto quiere decir que cuando se tiene una operación de conjunción primero se evalúa sólo el primer término: si su valor es falso, se sabe que toda la expresión será falsa y no tiene sentido seguir avanzando. Pero si el valor del término es verdadero, entonces se puede concluir que el valor de la expresión será igual al valor de la parte restante de la expresión.

or: Disyunción

La operación lógica de disyunción se expresa en Python usando la palabra reservada or. Cuando se tiene en un programa Python una expresión de la forma p or q se sabe entonces que tendrá un valor falso sólo si la expresión p y la expresión q tienen simultáneamente un valor falso. Además, el operador or es asociativo, así que se pueden escribir expresiones como w or x or y or z sin tener que usar paréntesis y sin riesgos de que cambie el resultado.

Pequeña diferencia

Al igual que con la operación and, Python también busca eficiencias en la evaluación de expresiones que usen en este operador. Esto se logra identificando inmediatamente que el resultado de una operación or entre el valor verdadero y cualquier valor, siempre será verdadero.

Es decir que, si el primer operando de una operación que use el operador or es verdadero, entonces el valor de la operación será verdadero y no será necesario evaluar los otros términos.

not: Negación

El tercer operador lógico en Python es la negación, la cual se expresa con la palabra reservada not. Como se trata de un operador unario se debe anteponer al operando.

El operador not no tiene ninguna diferencia con la operación lógica de negación y tiene una precedencia que es mayor a la de la conjunción y a la de la disyunción. Es decir que,

not a and b siempre será equivalente a (not a) and b y que a and not b siempre será equivalente a a and (not b).

Condicionales

if-else

En Python, las instrucciones condicionales se usan utilizando la siguiente estructura:

if expresion_booleana:
    cuerpo_if
else:
    cuerpo_else

expresion_booleana: es una expresión que tiene que tener un valor de verdad.

cuerpo_if: son las instrucciones que se van a ejecutar si la expresión booleana es verdadera en el momento de la evaluación.

cuerpo_else: son las instrucciones que se van a ejecutar si la expresión booleana es falsa en el momento de la evaluación.

if-elif-else

Ahora agregaremos la posibilidad de tener más de una condición, para lo cual tenemos que introducir los bloques que en Python se llaman elif.

if cond1:
    bloque1()
elif cond2:
    bloque2()
elif cond3:
    bloque3()
else:
    bloque_else()

Importante: Cuando se encuentra una condición verdadera, se ejecuta el cuerpo asociado a la condición y se termina la ejecución del condicional sin importar si más adelante había otras condiciones que también fueran verdaderas.

Diferencia: if-if y if-elif

Condicionales Anidados

Llamamos condicional anidado a la situación en la cual hay una instrucción condicional dentro del cuerpo de otra instrucción condicional.

def es_positivo_de_un_solo_digito_v1(x):
   if x > 0:
       if x < 10:
           return True
       else:
           return False
   else:
       return False

es_positivo_de_un_solo_digito_v1(25)
False

El mismo ejemplo, pero ocupando operadores booleanos:

def es_positivo_de_un_solo_digito_v2(x):
   if x > 0 and x < 10:
       return True
   else:
       return False

Recursividad

Introducción

En Python, la recursividad se refiere a una función que se llama a sí misma para resolver un problema.

Implica dos componentes críticos:

  • Caso base: Es la condición que pone fin a la recursividad. Sin ella, las llamadas recursivas continuarían eternamente.

  • Caso recursivo: Esta parte de la función consiste en descomponer el problema en partes más pequeñas y volver a llamar a la función con esas partes más pequeñas.

Puedes pensarlo así

Imagina una función que resuelve un problema complejo resolviendo versiones más pequeñas del mismo problema, llegando finalmente a una solución que no requiere más recursividad.

Ejemplo simple

El clásico ejemplo: EL FACTORIAL

\[ n! = \prod_{k=1}^{n}k = n \cdot (n-1) \cdot (n-2) \cdot ... \cdot 1 \] Por ejemplo: \[ 5! = 5\cdot4\cdot3\cdot2\cdot1 = 120 \]

En este caso, la propia definición matemática es recursiva:

\[ n! = n \cdot (n-1)! \]

lo que hace que la implementación del código sea intuitiva.

En Python

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

Como pueden notar, la función factorial() está incluida dentro de su definición.

Caso Base

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

Caso Recursivo

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

Vuelve a llamar a la función factorial()

Ejemplo iteración

def factorial(2):
    if 2 == 0:
        return 1
    else:
        return 2 * factorial(2-1)
def factorial(1):
    if 1 == 0:
        return 1
    else:
        return 1 * factorial(1-1)
def factorial(0):
    if 0 == 0:
        return 1
    else:
        return 0 * factorial(0-1)

Finalmente, el return de factorial(2)será:

return 2 * 1 * 1
factorial(2)
2

Problemas comunes

Aunque la recursividad puede ser una herramienta poderosa, es propensa a algunos problemas comunes, como:

  • Superar la profundidad máxima de recursión

Python tiene un límite de recursión por defecto de 1000 llamadas. Si una función recursiva se llama a sí misma demasiadas veces, aparecerá RecursionError.

  • Falta el caso base

Un caso base omitido o incorrecto hará que la función recurra infinitamente, provocando un desbordamiento de pila.
